中程度 — 特定条件で計算がズレる項目

4. セット商品のメガポ販売価格が特殊計算 対応済 2026-06-18

ツール: 定価 × 0.9 → セット = 8,064円

Excel: (定価 - 1,000) × 0.9 → セット = 7,164円

→ getSalePrice()でisSetフラグ判定、メガポの場合(定価-1000)×0.9を適用。手数料基準も(定価-1000)×5.9%に修正済み。

5. 50%OFF / 1000円OFFの対象範囲に制限なし 対応済 2026-06-18

50%OFF: わき用クリームに50%OFFは存在しない。TikTokにも50%OFFなし。

1000円OFF: セット商品2種のみ。

→ isDiscountValid()関数で制限判定。マトリクスでは「—」表示、ウォーターフォールでは警告バナー表示。クリニック/縁故は定価のみに制限。

軽微 — 端数・表示レベルの差異

6. 医師報酬の端数処理（Excel内の不整合）

HD報酬比較シート: floor(粗利×5%) を明記 → ローション = 121円

個別商品シート: 同じローション Shopify定価で 122円（ROUND使用の可能性）

ツールは Math.floor を使用しており、HD報酬比較シートと一致。個別シートとは最大1円の差異が出る場合がある。

7. Amazonクーポン150円の適用範囲

ツール: 10%OFF〜50%OFFに150円クーポンを加算。Excel: 同範囲（10%/20%/30%/50%OFF）に適用。概ね一致。ただし定価購入にはクーポンなし、わき用に50%OFFがないため実質的に問題なし。

初期投入: Supabase SQL Editorで直接INSERT（初回のみ）

HTML側: 起動時に4テーブルをPromise.all→P[]/C[]/OVERRIDES{}を動的生成。フォールバック付き。

効果: 原価変更時にHTML再編集不要。全値がSB「確定」扱いに。
