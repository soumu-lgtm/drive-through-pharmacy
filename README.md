# 薬品在庫管理システム プロトタイプ

ドライブスルー診療プロジェクト - データベース連携

## 概要

M3 DigiKar（電子カルテ）の薬品マスタをベースに、QRコードで入出庫を管理するシステムのプロトタイプです。

```
[M3 DigiKar] ──CSV──> [スプレッドシート] <──QR読取── [スマートフォン]
                           │
                           ├── 薬品マスタ
                           ├── 入庫履歴
                           ├── 出庫履歴
                           └── 在庫サマリー
```

## ファイル構成

```
proto/
├── index.html           # QR読み取りWebアプリ（メインUI）
├── qr_generator.html    # QRコードラベル生成ツール
├── gas_code.gs.txt      # Google Apps Script（スプレッドシート連携）
└── README.md            # このファイル
```

## セットアップ手順

### Step 1: スプレッドシートを作成

1. [Google スプレッドシート](https://sheets.google.com) で新規作成
2. スプレッドシート名: `薬品在庫管理_ドライブスルー診療`

### Step 2: Apps Script を設定

1. スプレッドシートの「拡張機能」→「Apps Script」を開く
2. `gas_code.gs.txt` の内容をコピー＆ペースト
3. 保存（Ctrl+S）
4. スプレッドシートをリロード
5. メニューに「在庫管理」が表示される

### Step 3: シートを初期化

1. メニュー「在庫管理」→「シートを初期化」をクリック
2. 以下の4シートが自動作成されます：
   - 薬品マスタ（青）
   - 入庫履歴（緑）
   - 出庫履歴（赤）
   - 在庫サマリー（黄）

### Step 4: サンプルデータ投入（テスト用）

1. メニュー「在庫管理」→「サンプルデータ投入」
2. 10種類の薬品サンプルが登録されます

### Step 5: QRコード生成

1. `qr_generator.html` をブラウザで開く
2. 「サンプルデータを読み込む」をクリック
3. 「QRコードを一括生成」をクリック
4. 「印刷する」で薬品棚用ラベルを印刷

### Step 6: Webアプリを公開（本番運用時）

1. Apps Script画面で「デプロイ」→「新しいデプロイ」
2. 種類: ウェブアプリ
3. アクセス: 全員（匿名ユーザーを含む）
4. デプロイURLをメモ

### Step 7: index.html を編集

`index.html` 内の以下の部分を、実際のApps Script URLに変更：

```javascript
// 現在はローカルデータを使用
// 本番時は以下のようにGASエンドポイントを設定
const GAS_URL = 'https://script.google.com/macros/s/xxxxx/exec';
```

## 使い方

### QR読み取りアプリ（index.html）

1. **モード選択**: 入庫（仕入れ）or 出庫（処方）
2. **カメラ起動**: QRコードをスキャン
3. **数量入力**: +/-ボタンで調整
4. **患者選択**（出庫時のみ）: 患者番号または氏名を入力
5. **記録**: ボタンを押して在庫を更新

### 在庫確認

- 下部タブの「在庫」で現在の在庫一覧を表示
- 赤字は発注点以下の薬品

### 履歴確認

- 下部タブの「履歴」で本日の入出庫履歴を表示

## M3 DigiKar からのデータ取り込み

### 材料マスタのCSVエクスポート

1. M3 DigiKar → 施設管理 → マスタ管理
2. 「材料」の「一覧」をクリック
3. CSVでエクスポート（※機能がある場合）

### 患者リストのCSVエクスポート

1. M3 DigiKar → 患者検索
2. 検索条件を設定（全件出す場合は条件なしで検索）
3. 「検索結果をCSVダウンロード」をクリック
4. ファイル名: `PatientList-YYYYMMDDHHMMSS.csv`

### 患者マスタのインポート手順

**方法1: Google Drive経由（推奨）**
1. M3からダウンロードしたCSVファイルをGoogle Driveにアップロード
2. スプレッドシートを開く
3. メニュー「在庫管理」→「患者マスタ」→「DriveのCSVからインポート」
4. ファイル名を入力（例: `PatientList-20260303172116.csv`）

**方法2: テキスト貼り付け（小規模データ向け）**
1. CSVファイルを開いてテキストをコピー
2. メニュー「在庫管理」→「患者マスタ」→「テキスト貼り付けでインポート」
3. 貼り付けてOKをクリック

### 患者ID自動補完

出庫履歴に患者名だけ入力して患者IDが空の場合、自動で補完できます。

**手動実行:**
- メニュー「在庫管理」→「患者マスタ」→「患者ID自動補完（手動実行）」

**自動実行（毎日朝9時）:**
- メニュー「在庫管理」→「患者マスタ」→「毎日9時の自動実行を設定」

**名前マッチングの仕様:**
- 空白（なし/半角/全角）を正規化して比較
- 例: 「山田太郎」「山田 太郎」「山田　太郎」はすべて同一人物として扱う
- 同姓同名や重複登録がある場合は、該当するIDをカンマ区切りで格納

### 手動での薬品登録

M3からのエクスポートができない場合：
1. スプレッドシートの「薬品マスタ」シートを開く
2. 手動で薬品情報を入力
3. 在庫サマリーにも初期在庫を登録

## トラブルシューティング

### カメラが起動しない

- HTTPS環境で実行してください
- ブラウザのカメラ権限を許可してください

### スプレッドシートに書き込めない

- Apps Script のデプロイ設定を確認
- 「全員がアクセス可能」になっているか確認

### QRコードが読み取れない

- 十分な明るさで読み取ってください
- カメラをQRコードに近づけすぎないでください

## デプロイ手順（手動）

### 問題: デプロイダイアログが読み込まれない場合

「デプロイを読み込んでいます...」が長時間続き、最終的にエラーになる場合：

1. **V8ランタイムが有効か確認**
   - Apps Script → プロジェクトの設定（⚙️）
   - 「Chrome V8 ランタイムを有効にする」がチェックされているか確認

2. **時間をおいて再試行**
   - Googleのサーバー側の問題の可能性があります
   - 数時間後に再度「デプロイ」→「新しいデプロイ」を試してください

3. **ブラウザのキャッシュクリア**
   - Ctrl+Shift+R でハードリフレッシュ
   - または別のブラウザで試す

### デプロイ成功後の設定

1. デプロイ完了後、URLをコピー（例: `https://script.google.com/macros/s/XXXXX/exec`）

2. `index.html` の以下の部分を編集：
```javascript
// 本番用GAS URLを設定
const GAS_URL = 'https://script.google.com/macros/s/XXXXX/exec';
const USE_GAS = true;  // trueに変更
```

3. index.htmlをWebサーバーまたはHTTPS環境でホスティング
