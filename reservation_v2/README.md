# クリニック予約システム プロトタイプ2

**佐川案（先日作成した既存モック＝プロトタイプ1）をベースに、湯浅案・西村案の長所を取り込んだ統合版。**

- **バックエンド/同期＝佐川案**：空き枠 =「診療時間マスタ − 予約」の引き算、予約が入った瞬間に受付ボードへリアルタイム反映（サブ秒）。本番は Supabase Realtime + RLS に差し替える継ぎ目（★SEAM★）を `store.js` に明示。
- **UI/画面遷移＝西村案**：クリニックカード選択 → 施術/枠選択（週間ストリップ）→ 2ステップ入力（確認画面＋同意モーダル）→ 完了（予約番号）。管理はダークヘッダー＋タブ＋日別ボード。
- **参考＝湯浅案**：4ステップの進捗表示・マイページ発想（今回は画面遷移の分かりやすさを主に参照）。

## ファイル
| ファイル | 役割 |

## 起動
プロジェクトルート（`C:\ClaudeWork`）でローカルサーバを立て、
`/projects/clinic_reservation_system/prototype2/demo.html` を開く。
（`file://` 直開きでも BroadcastChannel + localStorage で同期は動作）

## 実装済みの良い挙動（各案から継承）
- ダブルブッキング構造排除（残数チェック＋同一患者同時刻の二重予約ブロック）＝西村案の良い点
- 予約番号は `crypto.getRandomValues` を優先使用（西村案の指摘④＝`Math.random()`予測可能 を先回りで是正）
- リアルタイム同期のレイテンシ表示（佐川案の実測思想を継承）

## 本番化で差し替える点（★SEAM★）
- `store.js` の localStorage 読み書き → Supabase REST（**書き込みはサーバ関数経由**＝西村案の最大の穴①への対策）
- BroadcastChannel → Supabase Realtime（postgres_changes、legacy anon JWT）
