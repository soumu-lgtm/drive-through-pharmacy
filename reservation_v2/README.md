# クリニック予約システム

複数院（西春・横浜・千葉・中川）を統括する自前の予約システム。患者が予約すると受付ライブボードへ**リアルタイムで反映**されます。

- **バックエンド/同期**：空き枠 =「診療時間マスタ − 予約」の引き算で算出し、ダブルブッキングを構造的に排除。予約が入った瞬間に受付ボードへリアルタイム反映（サブ秒）。
- **UI/画面遷移**：クリニックカード選択 → 施術/枠選択（週間ストリップ）→ 2ステップ入力（確認画面＋同意モーダル）→ 完了（予約番号）。管理はダークヘッダー＋タブ＋日別ボード。
- Supabase 接続時は実DB＋Realtimeで同期。未接続時はローカル同期に自動フォールバック。

## ファイル
| ファイル | 役割 |
|---|---|
| `index.html` | 患者向けサイト（クリニック選択→施術→日時→入力→完了→照会/キャンセル） |
| `board.html` | 受付ライブボード（予約が即時フラッシュ表示） |
| `demo.html` | 左右並置デモ（患者⇄受付のリアルタイム同期を体感） |
| `store.js` | 同期ストア（マスタ−予約の空き枠エンジン＋Supabase/ローカル両対応） |
| `style.css` | スタイル |
| `supabase_setup.sql` | Supabase 用の専用デモテーブル作成SQL |

## 起動
プロジェクトルートでローカルサーバを立て、`demo.html` を開く。
（`file://` 直開きでも BroadcastChannel + localStorage で同期は動作）

## 実装済みの挙動
- ダブルブッキング構造排除（残数チェック＋同一患者同時刻の二重予約ブロック）
- 予約番号は `crypto.getRandomValues` で発行（予測されない）
- リアルタイム同期のレイテンシ表示

## 本番化で差し替える点（★SEAM★）
- `store.js` の Supabase 直接書き込み → サーバ関数経由（書き込みはサーバ側で権限判定）
- 同期は Supabase Realtime（postgres_changes）を利用
